# 全球实时地震监测大屏 PRD

## 目标

构建一个全屏沉浸式的全球实时地震监测大屏，用于观察全球近期地震事件的空间分布、震级强弱、实时动态和关键统计。

产品打开后直接进入监控界面：一张铺满视口的深色世界地图作为底座，真实地震事件以会呼吸的脉冲光圈落在地图上。左侧提供统计面板，右侧提供实时事件列表，顶部展示系统标题、在线状态、实时时钟和 LIVE 标识。

系统从 USGS 公开地震 GeoJSON 接口获取数据，无需用户登录、后端服务或 API Key。默认展示过去 24 小时数据，每 60 秒自动刷新。目标优先覆盖 1920x1080 桌面大屏，同时保证常见笔记本可用；移动端窄屏时面板可折叠或堆叠，但地图始终保持可见。

目标体验优先级：
1. 首屏先成立：地图铺满视口，深色、可拖动、可缩放。
2. 数据再点亮：真实地震点以脉冲光圈落到地图上，震级差异一眼可见。
3. 信息再完整：统计、列表、筛选、刷新、时间范围切换形成完整监控闭环。
4. 最后做质感：字体、网格、辉光、动画和响应式统一收口。

## 核心功能

1. 全屏深色世界地图
   - 地图占满浏览器视口，作为整个大屏的底层体验。
   - 使用深色地图瓦片或深色视觉方案。
   - 地图支持拖动、缩放。
   - 地图始终是视觉中心，左右面板不能遮挡主要地图观察区域。
   - 地图不是装饰预览，而是第一屏的主体验。

2. 地震数据接入
   - 使用 USGS Earthquake Hazards Program GeoJSON Feed。
   - 默认加载过去 24 小时全球地震数据。
   - 支持过去 1 小时、过去 24 小时、过去 7 天、过去 30 天时间范围。
   - 将 GeoJSON Feature 转换为应用内部地震事件模型。
   - 事件字段至少包含：震级、地点、发生时间、经纬度、深度、USGS 详情链接。
   - 请求失败时保留上一次成功数据，并展示清晰错误状态。

3. 脉冲地震标记
   - 每个地震事件在地图上显示为脉冲光圈标记。
   - 标记大小随震级增加而变大。
   - 震级越高，标记辉光越强、视觉优先级越高。
   - 点击标记展示弹窗，显示震级、地点、时间、深度和 USGS 详情链接。
   - 被选中的事件在地图上有更强的描边、辉光或层级突出。

4. 震级颜色分级
   - 小于 2.5：青绿色，轻微。
   - 2.5 至 4.4：绿色，较小。
   - 4.5 至 5.9：琥珀色，中等。
   - 6.0 至 6.9：橙红色，强震。
   - 大于等于 7.0：红色，重大地震。

5. 顶栏状态区
   - 左侧标题为 `GLOBAL SEISMIC MONITOR`。
   - 标题旁显示一个呼吸动画状态点。
   - 右侧显示每秒更新的实时时钟。
   - 右侧显示 `● LIVE` 标识。
   - 顶栏悬浮在地图上方，风格克制，不做厚重导航栏。
   - 标题使用科技感等宽或半等宽字体，例如 Chakra Petch。
   - 正文使用清晰无衬线字体，例如 Inter、system-ui 或同类字体。

6. 左侧统计面板
   - 展示当前可见事件总数。
   - 展示最大震级。
   - 展示平均震级。
   - 展示强震数量，例如大于等于 6.0 的事件。
   - 展示最近一次成功更新时间。
   - 展示自动刷新状态或下一次刷新倒计时。
   - 统计面板基于当前时间范围和当前震级筛选结果计算。

7. 右侧实时事件列表
   - 按发生时间倒序展示地震事件。
   - 每条事件展示震级、地点、发生时间和深度。
   - 使用震级颜色或标签辅助识别强弱。
   - 列表支持滚动浏览。
   - 点击列表行时，地图飞到对应地震位置，并打开或高亮对应事件。
   - 最新事件进入列表时可以有轻微动态反馈，但不能影响阅读和点击。

8. 震级筛选
   - 提供震级筛选条，支持按最低震级过滤。
   - 拖动筛选条后，低于阈值的弱震从地图和列表中消失。
   - 统计面板基于筛选后的可见事件计算。
   - 筛选过程应即时反馈，避免页面卡顿。
   - 默认最小震级为 0，展示当前时间范围内全部事件。

9. 自动刷新与手动刷新
   - 当前时间范围下每 60 秒自动刷新一次。
   - 支持手动刷新。
   - 刷新时不清空当前已有数据，避免大屏闪烁。
   - 刷新失败时保留上一次成功数据，并展示清晰错误状态。
   - 刷新成功后更新最近成功更新时间。

10. 时间范围切换
   - 支持过去 1 小时、过去 24 小时、过去 7 天、过去 30 天。
   - 切换后调用对应 USGS GeoJSON Feed。
   - 切换时展示轻量加载状态。
   - 切换完成后，地图、列表、统计和筛选结果同步更新。

11. 视觉与动效
   - 整体为深色科技感大屏。
   - UI 主色使用青色。
   - 震级色彩使用青绿色、绿色、琥珀色、橙红色、红色表达由低到高。
   - 背景叠加细网格纹理和角落微弱辉光。
   - 震级越高，标记辉光越强。
   - 首屏加载时，顶栏、左侧面板、右侧面板和关键控件使用 staggered 错峰淡入动画。
   - 动效克制，不能干扰地图拖动、缩放、定位和信息阅读。

12. 响应式行为
   - 1920x1080 下为完整大屏布局。
   - 常见笔记本宽度下保持左右面板和地图可用。
   - 移动端窄屏时面板可折叠或上下堆叠。
   - 无论屏幕宽度如何，地图必须始终可见。

## 明确不做

1. 不做用户登录、权限系统或个人账户。
2. 不做后端服务，数据直接从 USGS 公开接口读取。
3. 不做数据库持久化。
4. 不做地震预警、灾害预测、风险评估或安全决策建议。
5. 不做短信、邮件、桌面通知等主动告警。
6. 不做复杂 GIS 分析，例如板块边界、热力图、距离测量、历史轨迹对比、人口密度叠加等。
7. 不做多数据源融合，本阶段只使用 USGS GeoJSON。
8. 不做离线模式。
9. 不做营销落地页、产品介绍页或大段说明文案。
10. 不做移动端优先设计，移动端只保证基础可用和地图可见。

## 技术栈

1. 构建工具：Vite。
2. 前端框架：React。
3. 语言：TypeScript。
4. 样式：Tailwind CSS。
5. 地图：react-leaflet 与 Leaflet。
6. 数据源：USGS Earthquake Hazards Program GeoJSON Feed。
7. 字体建议：标题使用 Chakra Petch，正文使用 Inter、system-ui 或其他清晰无衬线字体。
8. 推荐接口映射：
   - 过去 1 小时：`https://earthquake.usgs.gov/earthquakes/feed/v1.0/summary/all_hour.geojson`
   - 过去 24 小时：`https://earthquake.usgs.gov/earthquakes/feed/v1.0/summary/all_day.geojson`
   - 过去 7 天：`https://earthquake.usgs.gov/earthquakes/feed/v1.0/summary/all_week.geojson`
   - 过去 30 天：`https://earthquake.usgs.gov/earthquakes/feed/v1.0/summary/all_month.geojson`
